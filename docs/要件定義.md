# PJ_NOTEBOOK 要件定義書

最終更新: 2026-07-20
対象システム: `outputs/system-design-basics-ja.html`（学習ジャーナル + 設計Wiki）

---

## 1. プロジェクト概要

### 1.1 背景
設計・開発の基礎知識を学ぶ過程で得た「気づき・メモ・疑問」は、その場では分類できないことが多い。
最初から整った分類を強制すると、記録そのもののハードルが上がり、学びが残らない。

### 1.2 目的
「まず気軽に書く → あとから整理する → 読み返す・検索する」を一連の流れとして支える、
ローカル完結・単一HTMLの**学習ジャーナル兼設計Wiki**を提供する。

### 1.3 コアコンセプト
- **人は最初から分類できない**を前提にする。未整理のまま残せることを重視。
- 「今日の記録」だけでなく、**読む / 書く / 整理する / 検索する / 振り返る**を等しく重視。
- あたたかい本棚カラー ＋ Notion風のWikiレイアウトの併用。
- ビルド不要・依存ライブラリなし・オフライン動作を守る。

## 2. 対象ユーザーと利用シーン

| 区分 | 内容 |
|---|---|
| 主対象 | 設計・開発を学習する個人（本システムの主開発者兼利用者） |
| 利用環境 | PCブラウザ中心。モバイルブラウザでも閲覧・記録可能 |
| 代表シーン | ①学習中に思いついたメモを即記録 ②あとでBook/章/タグへ整理 ③タスク化して進捗管理 ④用語や資料を読み返す ⑤検索で横断参照 |

---

## 3. スコープ

### 3.1 対象範囲（In Scope）
- 単一HTMLで完結する学習ジャーナル + Wikiアプリ。
- ブラウザの `localStorage` によるローカル保存。
- 同一PC内での簡易共有（ローカル同期サーバー `wiki-sync-server.mjs`）。
- JSONによるエクスポート／インポート／初期化。
- GitHub Pages 等での静的公開（閲覧・各ブラウザ内保存）。

### 3.2 対象外（Out of Scope／現状未実装）
- 本格的なログイン認証・アカウント管理。
- 複数人同時編集・リアルタイム共同編集。
- クラウド永続DB・サーバー側での恒久保存。
- 端末をまたいだ自動同期（別PC・別ネットワーク間）。

## 4. 用語定義（ドメインモデル）

| 用語 | 意味 |
|---|---|
| **本棚 (Shelf)** | Bookをまとめる最上位のグループ。切替・追加・改名・削除が可能 |
| **Book** | 章/Pageの集合。アイコン・色・短縮名・説明を持つ |
| **章 / Page** | 本文を持つ最小単位。`parentPageId` により階層（入れ子）を構成 |
| **ブロック (Block)** | Page本文を構成する要素（テキスト・見出し・ToDo・表など） |
| **record** | Wiki / Journal / Tasks / Dashboard / Database が共有する統一データ。`types`（複数値）を持つ |
| **type** | recordの分類。複数付与可（例: `memo`, `task`, `journal`, `question`, `term`, `note`） |
| **tag** | recordや本文に付与する横断ラベル |
| **Inbox** | 未整理recordの受け皿ページ |
| **今日の記録** | 当日分の学習ジャーナルページ（自動生成） |

## 5. 機能要件（機能一覧）

### 5.1 コンテンツ管理（本棚 / Book / 章）
- FR-01 本棚の作成・改名・削除・切替（`createShelf` / `renameShelf` / `deleteShelf` / `setActiveShelf`）
- FR-02 Bookの作成・改名・複製・削除（`renameBook` / `duplicateBook` / `deleteBook`）
- FR-03 Bookの短縮名・アイコン・色・説明の編集（`submitBookForm`）
- FR-04 章/Pageの作成・改名・複製・削除（`renamePage` / `duplicatePage` / `deletePage`）
- FR-05 Pageの階層化（親子・入れ子。`parentPageId` / `childPagesFor` / `descendantPageIds`）
- FR-06 Bookの折りたたみ表示（`toggleBook` / `isBookCollapsed`）
- FR-07 ドラッグによるBook・Page・ブロックの並び替え（`moveBookById` / `movePageById` / `moveBlock`）

### 5.2 記録の導線（書く）
- FR-08 空白から始める「今日の記録」自動生成（`ensureTodayJournalPage` / `openTodayJournal`）
- FR-09 気づきメモの即時追加（`addQuickMemo`）
- FR-10 未整理Inboxへの蓄積（`ensureInboxPage`）
- FR-11 「読む / 編集」モードの明示的切替（閲覧優先。`pageEditMode` / `canEditPageContent`）

### 5.3 ブロックエディタ（本文編集）
- FR-12 ブロック種別: Text / Heading / ToDo / Callout / Memo / Code / Math / Table / Database / DB Props / Synced / Backlink / File・Image
- FR-13 スラッシュコマンドによるブロック挿入（`processSlashCommand`）
- FR-14 ブロックの複製・削除・移動・種別変換（`duplicateBlock` / `deleteBlock` / `turnBlockInto`）
- FR-15 ブロックツールバーのsticky表示（編集中）
- FR-16 サマリーは色付きカードを1枚ごとにブロック化
- FR-17 添付（File / Image）の挿入（`insertAttachment`）
- FR-18 ブロックのプレビュー表示（`showBlockPreview`）

### 5.4 アノテーション（本文への注釈）
- FR-19 選択範囲へのアノテーション付与（`addAnnotationToSelection`）
- FR-20 アノテーションの色分け・メモ編集・色変更・削除（`editAnnotation` / `setAnnotationColor` / `removeAnnotation`）

### 5.5 表・Database
- FR-21 閲覧時の列ソート（`sortTableByColumn`）
- FR-22 編集時のセル編集・行/列の追加・削除・並び替え（`addTableRow` / `addTableColumn` / `moveTableColumn` / `deleteTableColumn` / `moveTableRow`）
- FR-23 表のフィルタ・グループ化（`filterTableByPrompt` / `groupTableByColumn`）
- FR-24 表のDatabase化・DBプロパティ管理（`convertTableToDatabase` / `refreshDbProperties` / `editDbProperty` / 型推論 `inferPropertyType`）

### 5.6 record・整理（あとから整理する）
- FR-25 メモ等をrecord化し、複数typeを付与（`ensureRecordForNote` / `addRecordTypes` / `removeRecordType`）
  - 例: `memo + task`、`journal + question + task`
- FR-26 ノートからPage作成・既存Pageへ移動・Bookへ移動（`createPageFromNote` / `moveNoteToExistingPage` / `moveNoteToBook`）
- FR-27 ノートのToDo化・DB化（`convertNoteToTodo` / `convertNoteToDb`）

### 5.7 共有ビュー（record横断）
- FR-28 共有Database表示（全recordを一覧・管理。`renderSharedDatabaseView`）
- FR-29 Tasksビュー: Todo / Doing / Done / Hold の状態別管理（`renderTasksView`）
- FR-30 Dashboard: 未整理・疑問・タスク・用語の状態を集計・俯瞰（`renderDashboardView`）
- FR-31 関連record・バックリンク・タグの表示（`renderRelatedRecords` / `pageBacklinks` / `pageTags`）

### 5.8 検索・ナビゲーション
- FR-32 全文検索と検索サジェスト（`searchResults`）
- FR-33 タグ・Book・章・本文への直接移動（`renderLinkedContent`）
- FR-34 hover / 右クリック / `...` メニューによる対象付近での操作（`showContextMenu` / `showBookManagementMenu` / `showPageManagementMenu` / `showNoteMenu`）
- FR-35 パンくず・クイックナビ・セクション折りたたみ

### 5.9 データ入出力・保存
- FR-36 `localStorage`（キー `systemDesignWiki.v1`）への自動保存（`saveState` / `persistState`）
- FR-37 ローカル同期サーバー経由の同一PC内共有（`loadRemoteState` / エンドポイント `http://127.0.0.1:8765/wiki-state`）
- FR-38 JSONエクスポート・インポート（`exportJson` / import入力）
- FR-39 初期状態へのリセット（`resetToInitialState` / `resetRemoteState`）
- FR-40 旧データからのマイグレーション（本棚・学習コピー・ワークスペースページ等の復元）

### 5.10 初期学習コンテンツ（シード）
- FR-41 以下の学習Bookを初期同梱: 設計 / 画面表示方式 / 開発言語 / DB / API・セキュリティ / インフラ・運用 / 品質 / 開発フロー / 拡張子・フォーマット / UI・UX
- FR-42 UI/UX本棚: EMOJI・フォント・GIF・Atomic Design・UIコンポーネント・バリアント・デザイントークンの資料
- FR-43 各種図の描画: ドキュメント/フロー/シーケンス/ER/ワイヤー/コンポーネント/インフラ/フォーマット（`renderFlowVisual` ほか `renderXxxVisual` 群）・用語解説（`renderTermExplanations`）

## 6. 非機能要件

### 6.1 アーキテクチャ・技術制約
- NFR-01 単一HTMLで完結（ビルド工程・外部依存ライブラリなし）。
- NFR-02 `file://` での直接オープン（ダブルクリック起動）を維持する。
- NFR-03 標準Web技術（HTML / CSS / Vanilla JS）のみで実装。

### 6.2 保存・データ
- NFR-04 通常はブラウザ `localStorage` に保存。ネット接続不要でオフライン動作。
- NFR-05 同期サーバー起動時は `outputs/wiki-state.json` にも保存し、同一PC内の別ブラウザと共有。
- NFR-06 個人の編集内容（`wiki-state.json`）はGit管理対象外（プライバシー）。
- NFR-07 データはJSONで可搬（エクスポート／インポートでバックアップ・移行可能）。
- NFR-08 旧バージョンのデータ構造からの後方互換マイグレーションを行う。

### 6.3 UI / UX
- NFR-09 通常時は「読む」ことを優先し、操作ボタンを常時出しすぎない。
- NFR-10 Page / 章 / Book / 本棚の編集モード・管理操作を混在させない。
- NFR-11 操作は対象の近くに置き、hover / 右クリック / `...` の内容を一致させる。
- NFR-12 あたたかい本棚カラー ＋ Notion風レイアウトの一貫性を保つ。
- NFR-13 Liquid Glass・`backdrop-filter`・過度なblurは使用しない。

### 6.4 レスポンシブ・アクセシビリティ
- NFR-14 モバイルでは本文を最優先。ページツリーはドロワー化。
- NFR-15 タップ領域は最低44px程度を確保。
- NFR-16 文字切れ・ボタン重なり・横方向はみ出しを起こさない。
- NFR-17 900px以下のブレークポイントでレイアウトを最適化。

### 6.5 性能
- NFR-18 記録・切替・検索が体感即時であること（ローカル処理・同期保存）。
- NFR-19 単一HTMLの規模増大に対し、描画・保存が実用速度を維持すること。

### 6.6 保守性
- NFR-20 スタイルは機能別CSSモジュール（`outputs/styles/*.css`）に分割し可読性を確保。
  ※CSS分割はリファクタ第1段階として実施済み。JavaScriptのモジュール分割は今後の課題。
- NFR-21 変更後は構文確認・ブラウザ表示・保存/再読込の回帰確認を行う。

### 6.7 公開・配布
- NFR-22 GitHub Pages で静的サイトとして閲覧可能（保存は各ブラウザの `localStorage`）。
- NFR-23 ローカル同期サーバーはGitHub Pages上では動作しない前提とする。

### 6.8 互換性
- NFR-24 モダンブラウザ（Chromium / Safari / Firefox 系）で動作すること。
